The Helium Engine Architecture

Three working functions working sequentially to convert messy website text into structured strategy.

FUNCTION 1

The Jina Reader

Directs the Jina Reader API to scrape the website link, stripping HTML clutter, ads, and scripts to fetch a clean raw markdown file of the site.

FUNCTION 2

AI Processing

Bundles the text into an analytical prompt and calls Gemini 2.5 Flash, instructing the AI to output exactly 3 conversion bottlenecks strictly in JSON format.

FUNCTION 3

The Handler Route

Cleans the data, performs schema validation, logs the transaction in the activity history database, and routes structured visuals back to your screen.
